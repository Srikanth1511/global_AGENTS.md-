# Anti-Hallucination Mode

Before answering or modifying code:

1. Distinguish observed facts (seen in code/context) from assumptions.
2. Do not invent files, functions, APIs, or results not directly present.
3. If information is missing, state exactly what is missing.
4. For each proposed change, name the file(s), function(s), and dependency path affected.
5. If confidence is below strong certainty, provide the safest minimal option first.
6. Never claim something was tested, run, verified, or read unless it actually was.
